form : 사용자가 입력한 데이터를 수집하기 위해 사용하는 태그 - input, texttarea, button, select, checkbox, radio, submit GET : 전송 URL에 입력 데이터를 쿼리 스트링 형식으로 보내는 것. - 속도는 굉장히 빠름 - 전송할 수 있는 데이터 수가 적음 - 데이터가 모두 노출되기 때문에 보안상 문제점이 있음. POST : 요청 바디에 담아 보내는 방식(사용자 눈에는 안보이는 곳) - 전송데이터는 노출X 하지만 속도가 느리다 GET방식은 데이터의 검색시 사용하면 유리 POST 방식은 중요한 데이터를 검색하거나 혹은 데이터의 추가/ 수정/ 삭제 등에 유리

form은 단독으로 사용하는 태그는 아님 input등이 다양한 태그들의 태그와 연계하여 사용 input : 사용자로부터 데이터를 입력받기 위해 사용되는 태그 - form 태그 내에 존재해야 입력 데이터를 전송할 수 있음. 웹 서버에 전송되는 데이터는 name 속성이 key값 value 속성이 value값 그리하여 keyvalue의 형태로 전송이됨.
